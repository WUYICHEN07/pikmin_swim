# 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
許多《皮克敏》(Pikmin) 玩家的運動方式並非以陸上步行或跑步為主，特別是針對水上運動愛好者（例如游泳選手或愛好者）。目前遊戲主要依賴步數來推動進度，導致這些玩家在進行水上運動時，無法將運動量有效轉換為遊戲進度。因此，我們希望開發一款系統，能將游泳等水上運動量轉換為《皮克敏》中的步數，讓水上運動玩家也能有完整的遊戲體驗。

### 目標用戶
- 《皮克敏》的現有玩家，且平時有進行水上運動（如游泳）習慣者。
- 擁有能記錄水上運動數據之智慧手錶（如 Apple Watch、Garmin 等）的運動員或運動愛好者。

### 核心價值主張
- **擴展遊戲場景**：讓玩家的水上運動努力不白費，將其精確轉換為遊戲內的步數進度。
- **個性化體驗**：提供與皮克敏類似的介面，並新增不同的水上/海洋主題背景供玩家切換使用。
- **無縫整合**：透過演算法將手錶記錄的游泳時長或划水頻率，公平且精確地轉換為等量步數。

---

## 2. 功能需求

### 2.1 步數核心轉換演算法（核心）
- **作為** 水上運動玩家，
- **我希望** 系統能讀取我輸入的游泳時長或划水頻率，
- **以便** 系統能利用演算法將其轉換成對應的陸上步數。

### 2.2 運動紀錄儲存與查詢
- **作為** 玩家，
- **我希望** 我每次轉換後的步數紀錄都能被系統儲存，
- **以便** 我能隨時查看過去的運動量和對應獲得的皮克敏步數。

### 2.3 個性化背景切換
- **作為** 玩家，
- **我希望** 在查看運動步數介面時能更換不同的海洋/水上主題背景，
- **以便** 我能獲得更具沉浸感的視覺體驗。

### 2.4 使用者登入與註冊（選配/基本）
- **作為** 系統使用者，
- **我希望** 能夠建立並登入自己的帳號，
- **以便** 我的運動紀錄與背景設定能被安全地保存且不會與他人混淆。

### 2.5 數據儀表板與視覺化
- **作為** 玩家，
- **我希望** 能在首頁看到我累積的總轉換步數與今日轉換步數的圖表，
- **以便** 我能清楚掌握自己的整體運動成效。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架進行開發。
- **前端模板**：使用 Jinja2 模板引擎進行頁面渲染，結合 HTML/CSS。
- **資料庫**：使用 SQLite 作為關聯式資料庫，儲存使用者資料與運動紀錄。

### 效能與安全考量
- 轉換演算法需在合理時間內（< 1秒）返回結果。
- 使用者密碼需經過雜湊（Hash）處理後才存入 SQLite 資料庫。
- 確保基本防範 SQL Injection 與 XSS 攻擊。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 基本的 Web 介面（Flask + Jinja2）。
- 步數核心轉換邏輯（接收游泳時長/划水頻率，輸出步數）。
- 將轉換紀錄寫入 SQLite 資料庫並能在畫面上顯示歷史紀錄。

### Should Have (應該具備)
- 個性化背景切換功能（提供至少2-3種背景選項供玩家選擇）。
- 基礎的使用者帳號系統（註冊與登入）。

### Nice to Have (有了更好)
- 自動串接智慧手錶 API 直接獲取運動數據（初期MVP可先用手動輸入數據測試）。
- 累積步數的圖表視覺化展示。
- 成就系統（例如達到特定累積游泳步數解鎖特殊背景）。
