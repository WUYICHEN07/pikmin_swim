# PRD (Product Requirements Document)

## 專案名稱
皮克敏水性類型運動換算步數系統

## 1. 專案概述
- **背景與動機**：許多《Pikmin Bloom》玩家的運動習慣以水上運動（如游泳）為主，而非傳統的陸上步行。為了讓這些玩家也能享受遊戲樂趣，並將運動成果轉換為遊戲內的步數，我們希望開發一個第三方紀錄與步數換算系統，藉此增加遊戲體驗感與參與感。
- **目標用戶**：有進行水上運動習慣的《Pikmin Bloom》玩家、水上運動選手（擁有水上運動手錶的使用者）。
- **核心價值主張**：透過連接水上運動手錶（或手動紀錄），將游泳等水上運動的運動量轉換為對應的步數紀錄，並提供類似《Pikmin Bloom》可自訂背景的個人化介面，讓水上運動者也能擁有滿滿的成就感。

> ⚠️ **聲明**：本專案僅做為第三方紀錄與模擬驗證工具，**不包含**直接破解任天堂《Pikmin Bloom》伺服器進行官方數據寫入。

---

## 2. 功能需求

以下列出系統的核心主要功能及使用者故事：

### 2.1 運動紀錄匯入與新增
- **使用者故事**：作為一名水上運動玩家，我希望可以手動輸入運動數據（如游泳時間、距離），或未來能匯入手錶數據，以便系統能取得我的運動量。

### 2.2 步數轉換引擎
- **使用者故事**：作為系統使用者，我希望系統能根據特定的演算法，自動將我的水上運動數據轉換為等效的步數，以便我清楚知道今天相當於走了多少步。

### 2.3 歷史紀錄查詢
- **使用者故事**：作為使用者，我希望能夠在系統上查看過去所有的運動紀錄與步數轉換歷史，以便追蹤我的長期運動成果。

### 2.4 個人化背景介面
- **使用者故事**：作為玩家，我希望系統擁有類似皮克敏的親切介面，並且允許我更換不同的背景主題，以便我能依據心情客製化視覺體驗。

### 2.5 資料庫與使用者管理 (CRUD)
- **使用者故事**：作為系統開發者與使用者，我希望系統能穩定且安全地儲存使用者資訊、運動紀錄、步數轉換紀錄（支援完整的增刪改查 CRUD 操作），以便資料不遺失且易於管理。

---

## 3. 非功能需求

- **技術限制**：
  - 後端：Python Flask
  - 前端渲染：Jinja2 模板引擎
  - 資料庫：SQLite（輕量級關聯式資料庫）
- **效能與安全考量**：
  - 確保使用者的運動數據隱私不外洩。
  - 歷史紀錄查詢必須有良好的效能，即使累積大量紀錄也能迅速讀取。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須擁有)
- 使用者基本登入/註冊功能。
- 建立 SQLite 資料表結構（包含使用者、運動紀錄、步數轉換紀錄）並完成 CRUD 函數。
- 手動新增水上運動紀錄（距離、時間）的表單介面。
- 基本的步數轉換邏輯（例如：100公尺游泳 = N 步）。
- 歷史紀錄查詢與列表呈現頁面。

### Should Have (應該擁有)
- 允許使用者從幾種預設背景中選擇並更換（類似皮克敏介面風格）。
- 簡單的個人運動數據總覽或統計。

### Nice to Have (有了更好)
- 直接串接特定品牌運動手錶的 API 自動匯入數據。
- 達成特定步數的勳章或成就系統。

# 產品需求文件 (PRD) - 皮克敏水性類型運動換算步數系統

## 1. 專案概述

### 背景與動機
目前市面上的《Pikmin Bloom》（皮克敏）遊戲主要透過陸上步數來進行遊戲與角色成長，這使得以水上運動（如游泳）為主的玩家無法有效地將他們的運動量轉換為遊戲進度。為了解決這個問題，並增加水上運動愛好者的遊戲體驗，本專案旨在開發一個「水性類型運動換算步數系統」。此系統不直接破解官方伺服器，而是作為第三方紀錄與模擬驗證平台，將運動手錶記錄的游泳數據轉換為等效步數，並呈現皮克敏的成長狀態。

### 目標用戶
《Pikmin Bloom》的現有玩家，尤其是經常從事水上運動（如游泳）並配戴運動手錶的玩家。

### 核心價值主張
- 將水上運動數據（如游泳距離、時間）合理轉換為虛擬步數。
- 提供與《Pikmin Bloom》風格相仿的個人運動成就看板，即時展示數據轉換成果與皮克敏成長狀態。
- 支援不同背景設定，滿足玩家的個人化需求，增加遊戲沉浸感。

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事（User Story）呈現：

1. **個人運動成就看板展示（核心功能）**
   - 作為玩家，我希望在個人運動成就看板看到即時的游泳數據、轉換步數及進度條，以便隨時掌握目前的運動成果與皮克敏成長狀態。
2. **皮克敏成長模擬與成就解鎖**
   - 作為玩家，我希望看到累積的轉換步數能讓系統內模擬的皮克敏成長或解鎖特殊成就，以便獲得成就感與遊戲樂趣。
3. **運動數據輸入與換算**
   - 作為水上運動玩家，我希望能夠輸入或匯入手錶的游泳數據，以便系統幫我自動依據設定的比例換算成皮克敏步數。
4. **個人化背景切換**
   - 作為玩家，我希望系統提供不同風格的背景供選擇與切換，以便配合不同的運動情境或個人喜好，豐富視覺體驗。
5. **歷史紀錄查詢**
   - 作為玩家，我希望可以查詢過去的運動紀錄與轉換歷史，以便追蹤長期的運動與皮克敏成長進度。

## 3. 非功能需求

### 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite
- 前端技術：使用 HTML/CSS/JavaScript，並透過 Fetch API 進行非同步的資料串接。

### 效能與安全考量
- **效能**：確保 Fetch API 取得資料的速度流暢，進度條與數據更新的延遲最小化，提供平順的使用者體驗。
- **安全與隱私**：確保使用者上傳的運動數據具有基本的隱私保護。
- **合規性**：系統明確標示為第三方數據轉換與模擬驗證工具，絕不涉及任天堂官方《Pikmin Bloom》伺服器的破解或非法寫入。

## 4. MVP 範圍

### Must Have (必須擁有)
- 個人運動成就看板介面（包含數據展示、動態進度條）。
- 前端與後端的 Fetch API 數據串接（即時更新看板資料）。
- 基礎的游泳數據換算邏輯（將輸入的游泳數據轉換為步數）。
- 基本的皮克敏成長狀態圖示切換與顯示。

### Should Have (應該擁有)
- 成就解鎖畫面與觸發機制。
- 多種背景主題的切換功能。
- 運動紀錄的歷史檢視清單。

### Nice to Have (可以擁有)
- 直接與特定廠牌運動手錶 API（如 Garmin, Apple Watch）進行自動同步。
- 社交分享功能，將個人成就看板分享至社群媒體。
