# 產品需求文件 (PRD)：皮克敏水性類型運動換算步數系統

## 1. 專案概述
- **背景與動機**：許多皮克敏 (Pikmin Bloom) 玩家的運動方式並非全為陸上運動（如走路、跑步），許多水上運動選手或愛好者無法透過現有的步數系統獲得對應的遊戲回饋。為了解決這個問題，希望能開發一套第三方系統，將水上運動的紀錄換算為步數，提升這類玩家的遊戲體驗感。
- **目標用戶**：皮克敏玩家，特別是從事水上運動（如游泳、水球、潛水等），且配戴支援運動紀錄的穿戴裝置（運動手錶）的用戶。
- **核心價值主張**：提供一個介接水上運動數據並將其合理轉換為「步數」的模擬與紀錄系統，並結合不同背景設計，讓水上運動愛好者也能充分享受皮克敏遊戲的核心樂趣。

## 2. 功能需求
作為 MVP，我們至少需要以下主要功能：

1. **F-01 運動數據接入 API 整合 (核心功能)**
   - **作為**：穿戴裝置（或數據模擬器）
   - **我希望**：能透過 API 發送 JSON 格式的運動數據（如運動類型、時長、距離、心率等）至後端
   - **以便**：後端能接收、驗證數據格式，並為後續的步數換算做準備。
   *(註：此為您本次特別指定的負責功能)*

2. **F-02 水上運動步數換算邏輯**
   - **作為**：系統後端
   - **我希望**：能根據不同的水上運動數據（如游泳 1000m），依照自訂公式換算成合理的皮克敏走路步數
   - **以便**：玩家的努力能獲得公平的數值回報。

3. **F-03 個人數據總覽與紀錄展示**
   - **作為**：玩家
   - **我希望**：能在 Web 介面上看到我的歷史換算紀錄與累積「步數」
   - **以便**：追蹤我的運動成果與遊戲進度。

4. **F-04 個人化背景切換**
   - **作為**：玩家
   - **我希望**：能在前端介面切換不同的水系背景（如不同的水域風格、水上主題）
   - **以便**：提升使用體驗與代入感，符合水性運動的特色。

5. **F-05 使用者註冊與系統登入**
   - **作為**：玩家
   - **我希望**：能建立專屬帳號並登入系統
   - **以便**：將我的穿戴裝置數據與我個人的紀錄安全地綁定在一起。

## 3. 非功能需求
- **技術限制**：後端採用 Flask，前端畫面由 Jinja2 渲染，資料庫使用輕量級的 SQLite。
- **邊界與排除**：本次專案僅做第三方紀錄與模擬驗證，**絕不包含**直接破解或寫入任天堂《Pikmin Bloom》官方伺服器。
- **資料格式**：API 收發資料以 JSON 格式進行。
- **效能與安全**：需具備基礎的 API 資料驗證邏輯，防止格式錯誤或惡意數據寫入系統。

## 4. MVP 範圍
- **Must Have (必須有)**
  - F-01 運動數據接入 API (含基礎 JSON 驗證)
  - F-02 水上運動步數換算邏輯
  - F-03 個人數據展示與紀錄列表
- **Should Have (應該有)**
  - F-04 個人化背景切換機制
  - 基礎的使用者帳戶綁定
- **Nice to Have (加分項)**
  - 更精緻的動態海洋/水系特效背景
  - 穿戴裝置的真實 OAuth API 串接（MVP 階段可用模擬腳本替代測試）
