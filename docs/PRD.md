# 產品需求文件 (PRD)：皮克敏水性類型運動換算步數系統

## 1. 專案概述
- **背景與動機**：許多《Pikmin Bloom》（皮克敏）玩家的運動方式為水上運動（如游泳、水球、衝浪等），而非傳統的陸上步行。為了讓這些玩家也能將他們的運動量轉換為遊戲內的步數，並增加水下情境的遊戲體驗，開發此第三方紀錄與模擬驗證系統。
- **目標用戶**：喜愛水上運動的皮克敏玩家。因為許多玩家的運動方式不是以陸上運動呈現，希望可以連接至水上類選手們的運動手錶來轉換走路步數。
- **核心價值主張**：提供一個介面與原版皮克敏相似的系統，能記錄水上運動數據，將游泳等水性運動量換算為步數，並搭配不同背景與 AI 生成的專屬水下場景素材，大幅提升玩家的客製化與遊戲體驗感。

## 2. 功能需求
1. **運動數據紀錄與手錶連結**
   - 作為 **水上運動玩家**，我希望 **能將記錄方式連接至我的運動手錶**，以便 **自動匯入我的水上運動數據（如游泳距離、時間等）**。
2. **水上運動至步數換算**
   - 作為 **水上運動玩家**，我希望 **系統能將我的水上運動數據換算為步數**，以便 **我能累積等價的步數用於模擬培育皮克敏**。
3. **自訂水下主題背景與 UI**
   - 作為 **重視視覺體驗的玩家**，我希望 **能新增並切換不同的背景（如水下場景）**，以便 **供不同玩家使用，讓介面更符合我的水上運動情境**。
4. **水下皮克敏素材展示**
   - 作為 **皮克敏收集者**，我希望 **能看到符合水下主題的皮克敏插圖素材**，以便 **增加整體的遊戲體驗感與收集的樂趣**。
5. **第三方運動紀錄與模擬面板**
   - 作為 **系統使用者**，我希望 **能有一個儀表板檢視我的歷史轉換紀錄與模擬累積的步數**，以便 **追蹤我的運動成果（本次不包含直接破解任天堂伺服器進行官方數據寫入，僅做第三方紀錄與模擬驗證）**。

## 3. 非功能需求
- **技術限制**：後端預設採用 Flask + Jinja2 模板引擎與 SQLite 資料庫進行開發（依循專案常規配置）。
- **UI/UX 設計**：前端介面需高度美化，整合 AI 圖像生成工具產出的水下場景與素材，呈現高質感的視覺效果。
- **效能與安全考量**：
  - 確保使用者運動數據的隱私安全。
  - 前端大量圖片素材需經過優化，確保網頁載入流暢。

## 4. MVP 範圍
- **Must Have (必須有)**
  - 基礎運動數據紀錄功能（支援手動輸入或初步手錶資料匯入驗證）。
  - 水上運動至步數的轉換邏輯與計算引擎。
  - 結合 AI 生成素材的前端儀表板與水下場景 UI 佈局。
- **Should Have (應該有)**
  - 更多樣的背景切換功能供不同玩家使用。
  - 運動數據歷史紀錄圖表或清單檢視。
- **Nice to Have (可以有)**
  - 自動化串接市面上常見運動手錶（如 Garmin, Apple Watch）的官方 API。
  - 豐富的前端微動畫與互動效果。
